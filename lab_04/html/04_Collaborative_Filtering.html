Collaborative filtering¶

La idea del filtrado colaborativo es considerar las opiniones de los usuarios sobre las distintas pelculas y recomendar la mejor pelcula a cada usuario basndose en las clasificaciones anteriores del usuario y en la opinin de otros tipos de usuarios similares.

Este tipo de filtro se basa en la combinacin del comportamiento del usuario y en la comparacin y el contraste con el comportamiento de otros usuarios en la base de datos. La historia de todos los usuarios juega un papel importante en este algoritmo. La principal diferencia entre el Content Based Filtering y el Collaborative filtering es que en este ltimo, la interaccin de todos los usuarios con los elementos influye en el algoritmo de recomendacin, mientras que para el filtrado basado en el contenido slo se tienen en cuenta los datos del usuario afectado. Hay mltiples formas de aplicar el filtrado en colaboracin, pero el concepto principal que hay que captar es quelos datos de mltiples usuarios influyen en el resultado de la recomendacin y no dependen de los datos de un solo usuario para la modelizacin.

Se distinguen dos tipos de collaborative filtering:

  • User-based Collaborative filtering
  • Item-based Collaborative Filtering

User-User collaborative filtering¶

Encuentra puntuaciones de similitud entre los usuarios para elegir a los usuarios ms similares y recomienda productos que estos usuarios similares han gustado o comprado anteriormente.

En el caso de las pelculas, este algoritmo encuentra la similitud entre cada usuario basndose en las puntuaciones que han dado previamente a diferentes pelculas. La prediccin de un artculo para un usuario u se calcula calculando la suma ponderada de las puntuaciones de usuario dadas por otros usuarios a un artculo i. As, la prediccin para el usuario 'u' se da como:

P(u,i) = E [r(v,i) * s(u,v)] / E [S(u,v)]

donde,

Pu, i es la prediccin de un elemento Rv, i es la calificacin dada por un usuario v a una pelcula i Su,v es la similitud entre los usuarios

Pasos bsicos para esto:

Para las predicciones necesitamos la similitud entre el usuario u y v. Podemos hacer uso de la correlacin de Pearson. Primero encontramos los elementos calificados por ambos usuarios y en base a las calificaciones, se calcula la correlacin entre los usuarios. Las predicciones pueden ser calculadas usando los valores de similitud. Este algoritmo, en primer lugar calcula la similitud entre cada usuario y luego basado en cada similitud calcula las predicciones. Los usuarios que tengan una mayor correlacin tendern a ser similares. Desventaja: este algoritmo requiere bastante tiempo, ya que implica el clculo de la similitud para cada usuario y luego calcula la prediccin para cada puntaje de similitud. Una forma de manejar este problema es seleccionar slo unos pocos usuarios (vecinos) en lugar de todos para hacer predicciones, es decir, en lugar de hacer predicciones para todos los valores de similitud, elegimos slo unos pocos valores de similitud. Hay varias maneras de seleccionar los vecinos:

  • Seleccionar un umbral de similitud y elegir todos los usuarios por encima de ese valor
  • Selecciona al azar a los usuarios
  • Ordena los vecinos en orden descendente de su valor de similitud y elige los usuarios top-N
  • Usar la agrupacin para elegir los vecinos

Este algoritmo es til cuando el nmero de usuarios es menor. No es efectivo cuando hay un gran nmero de usuarios, ya que llevar mucho tiempo calcular la similitud entre todos los pares de usuarios. Esto nos lleva a un filtrado colaborativo item-item, que es efectivo cuando el nmero de usuarios es mayor que los artculos recomendados.

Item-Item collaborative filtering¶

Se encuentran similitudes entre cada uno de las pelculas y en base a eso, se hacen recomendaciones de pelculas similares para el usuario.

Este algoritmo funciona de manera similar al filtrado colaborativo usuario-usuario con slo un pequeo cambio: en lugar de tomar la suma ponderada de las clasificaciones de "user-neighbors", tomamos la suma ponderada de las clasificaciones de "item-neighbors".

Este mtodo presenta un problema de escalabilidad y falta de de algunos datos. En casos extremos, podemos tener millones de usuarios y la similitud entre dos pelculas bastante diferentes podra ser muy alta simplemente porque tienen un rango similar para el nico usuario que las clasific a ambas.

Una forma de manejar el problema de la escalabilidad y la escasez de datos creado por el filtro, es aprovechar un modelo de factores latentes (las variables latentes o variables ocultas, son las variables que no se observan directamente sino que son inferidas a travs de un modelo matemtico a partir de otras variables que se observan) para calcular la similitud entre los usuarios y los artculos.

Esencialmente, queremos convertir el problema de recomendacin en un problema de optimizacin. Podemos verlo como lo buenos que somos en la prediccin de la calificacin de los artculos dados a un usuario. Una mtrica comn es el error cuadrado medio de la raz (RMSE). Cuanto ms bajo es el RMSE, mejor es el rendimiento.

En lugar de implementar un filtrado colaborativo desde cero, utilizaremos la librera Surprise que utiliza algoritmos extremadamente poderosos como la Descomposicin del Valor Singular (SVD) para minimizar el RMSE (Root Mean Square Error) y dar buenas recomendaciones

Las ventajas de este mtodo de recomendacin es que no necesita conocer caratersticas de las pelculas como el gnero ya que utiliza la informacin recopilada de otros usuarios para recomendar nuevos elementos al usuario actual. Sin embargo, no logra recomendar una nueva pelcula que no tienen clasificaciones. Requiere de la comunidad de usuarios y puede tener un problema de escasez de datos.

Conclusiones generales:¶

En este proyecto hemos implementado distintos algoritmos de recomendacin aplicados a una base de datos con informacin acerca de pelculas. Estos han sido:

  • Sistema de recomendacin demogrfico. Este es el sistema ms simple de todos, ya que ofrece recomendaciones generalizadas a los usuarios basadas en la popularidad de la pelcula y/o el gnero, recomendando lo mismo a aquellos usuarios con caractersticas demogrficas similares.
  • Sistema de recomendacin basado en el contenido. Para este sistema hemos elaborado dos algoritmos: uno basado en la descripcin de la pelcula y otro basado en un conjunto de metadatos (director, actores, gneros relacionados y palabras clave de la trama).
  • Sistema de recomendacin colaborativo. Este algoritmo se caracteriza por recomendar artculos en base a las opiniones de los usuarios similares. Aunque este sistema es de gran utilidad, tiene la desventaja de requerir que la comunidad de usuarios punte las pelculas.

A travs de la elaboracin de los sistemas de recomendacin, se resuelve uno de los grandes problemas de la actualidad: el exceso de informacin. De esta manera, los usuarios tienen recomendaciones personalizadas para que consuman aquellos productos ms afines a sus gustos.

Mediante la realizacin de este trabajo hemos sido conscientes del conflicto entre la tica y el negocio que puede suponer la implementacin de este tipo de algoritmos. Esto es debido a que grandes empresas como Netflix, Amazon o Spotify son capaces de generar unos beneficios muy altos al disponer de una gran cantidad de informacin sobre las preferencias de sus clientes, sin (en muchos casos) ser estos conscientes.

Adicionalmente, querramos destacar la importancia de la extraccin, procesamiento y estructuracin de los datos para poder trabajar con ellos de manera eficiente.
